# Elementwise 算子

Elementwise 算子表示对数组中对应位置的元素分别进行计算。当前实现的是逐元素加法：

[
C[i] = A[i] + B[i]
]

其中，输入数组 `A` 和 `B` 的长度相同，输出数组 `C` 保存对应元素相加的结果。

## 版本说明

### elementwise_v0

最朴素的实现方式。

* 一个线程负责一个元素。
* 使用全局线程编号定位元素。
* 相邻线程访问相邻数据，可以实现合并访存。

```cpp
C[idx] = A[idx] + B[idx];
```

### elementwise_v1

使用 Grid-Stride Loop。

* 一个线程可以处理多个元素。
* 每次循环的步长为整个网格中的线程总数。
* 可以使用固定数量的线程处理任意长度的数组。

```cpp
for (int i = idx; i < num_elements; i += stride)
{
    C[i] = A[i] + B[i];
}
```

### elementwise_v2

使用 `float4` 进行向量化访存。

* 一个线程连续处理四个 `float` 元素。
* 每次读取或写入一个 `float4`，即 16 字节数据。
* 减少线程数量和索引计算次数。
* 需要单独处理元素数量不能被 4 整除的情况。

### elementwise_v3

结合 Grid-Stride Loop 和 `float4` 向量化访存。

* 每个线程通过网格跨步循环处理多个 `float4`。
* 同时具备 Grid-Stride Loop 的通用性和向量化访存的特点。
* 最后单独处理不足四个的剩余元素。

## 版本对比

| 版本 | 实现方式                        | 每个线程处理的数据        |
| -- | --------------------------- | ---------------- |
| v0 | 普通逐元素计算                     | 1 个 `float`      |
| v1 | Grid-Stride Loop            | 多个间隔元素           |
| v2 | `float4` 向量化                | 连续 4 个 `float`   |
| v3 | Grid-Stride Loop + `float4` | 多组连续 4 个 `float` |

Elementwise 加法的计算量很小，通常属于显存带宽受限算子，因此优化重点主要是减少访存指令、降低线程和索引开销，并尽可能提高显存访问效率。
